# 澪音（みおん）公式サイト 要件定義書

## 1. 背景・目的
- バーチャルシンガー「澪音（みおん）」の世界観・音楽活動を統合的に発信する公式サイトを構築する。
- Suno で公開している楽曲への導線を強化し、視聴・拡散を促す。
- ファンコミュニティやライブ情報など、継続的な活動フォローができるプラットフォームを提供する。

## 2. サイト概要
- **URL**: （未定）
- **公開環境**: 静的ホスティング（GitHub Pages など）を想定。
- **想定端末**: PC、タブレット、スマートフォン。
- **想定ページ構成**: 単一ページのランディングサイト（スクロール型）。

### 2.1 ターゲットユーザー
1. 澪音の既存ファン（最新情報の収集・コミュニティ参加が目的）。
2. Suno などで楽曲を知った新規リスナー（プロフィールや世界観の理解、視聴が目的）。
3. コラボレーションやメディア露出を検討するビジネスパートナー（活動実績やコンタクト情報が目的）。

### 2.2 ブランド・トーン&マナー
- キーワード: 「透明感」「未来感」「夜明け」「幻想的」「洗練」。
- ビジュアル: グラデーション、ガラスモーフィズム、柔らかな光表現を取り入れる。
- 言語: 丁寧で落ち着きのある日本語。専門用語は控え、余白とリズムを意識したコピー。

## 4. 機能要件
1. **アンカーリンクによるスムーズスクロール**: ナビゲーションリンクから該当セクションへスムーズスクロールを実装。
2. **レスポンシブ対応**: 768px、1024px 付近でのレイアウト最適化。モバイルでは縦スクロール主体、PCではグリッド活用。
3. **CTAボタンの視覚的強調**: Suno リンクボタンをプライマリカラーで強調。
4. **外部リンク挙動**: Sunoなど外部サービスへ遷移するリンクは `target="_blank"` と `rel="noopener"` を設定。
5. **アイコン/ロゴ表示**: 提供されたアイコン画像をロゴおよびファビコンとして使用（`assets/` 配下に格納）。
6. **メタ情報設定**: `meta description`、OGP 用メタタグ（任意）を設定し、SNSシェア時の訴求力を高める。
7. **将来的な拡張余地**: ニュース/ライブ情報追加、埋め込みプレイヤー、フォーム導入を想定したセクション構造にする。

## 5. デザイン要件
- **レイアウト**: CSS Grid・Flexbox を用いてセクションごとに余白と視線誘導を設計。
- **カラー**: 夜明けを想起させるブルー〜パープル系グラデーションを基調。アクセントに温かみのあるピンク/オレンジを差し込む。
- **タイポグラフィ**: 見出しはサンセリフ体、本文は可読性の高い汎用日本語フォントスタックを使用。行間を広めに確保。
- **ビジュアル要素**: ガラスモーフィズム風のカード、淡いぼかし、浮遊感のある装飾（オーブ、ライン）を適用。
- **アニメーション**: ホバー時の軽微なトランジション、ヒーロー背景のわずかなモーションなどパフォーマンスを損ねない範囲で実装。
- **アクセシビリティ**: カラーコントラスト比 WCAG AA 以上を目標。キーボードフォーカスの見える化。

## 6. 技術要件
- **使用技術**: 最新仕様に準拠したプレーンな HTML5 + CSS3。JavaScript は必須ではないが、スムーズスクロール等で必要な場合のみ最小限導入可。
- **構成**: `index.html`、`styles.css`、`assets/`（画像・アイコン）のシンプル構成。
- **命名規則**: BEM をベースにしたクラス名。CSS カスタムプロパティを活用しテーマカラーや余白を管理。
- **パフォーマンス**: 不要な外部ライブラリを極力排除し、画像は最適化された SVG/PNG を使用。
- **互換性**: 最新版の主要ブラウザ（Chrome, Safari, Firefox, Edge）の現行バージョンで表示崩れがないこと。

## 7. コンテンツ更新・運用
- GitHub リポジトリでバージョン管理し、Pull Request ベースで更新する。
- 新曲リリースやイベント発生時は、音楽セクションおよびタイムラインを更新。
- SNS アカウントの変更や追記はコミュニティセクションに反映。

## 8. 品質要件
### 8.1 アクセシビリティ
- `lang="ja"` の設定、画像の `alt` 属性、構造化された見出し階層を維持。
- タブ操作で主要コンテンツへ到達できるようフォーカス管理。

### 8.2 表示パフォーマンス
- ファーストビュー内の主要要素は遅延なく表示。Lighthouse パフォーマンススコア 80 以上を目標。
- CSS・画像の軽量化を継続的に確認。

### 8.3 SEO
- 適切な `title`、`meta description` を設定し、OGP 画像（任意）を用意。
- セクション見出しは `h1`～`h3` を構造的に利用し、検索エンジンに内容を伝える。

4. テスト（表示確認、アクセシビリティチェック）: 3日
5. 公開準備・リリース: 2日
